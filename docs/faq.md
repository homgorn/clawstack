# FAQ

## Is this repo usable without a paid service?

Yes. That is the point of the open-core model recommended here.

## What is the paid layer for then?

Speed, convenience, support, hosting, monitoring, and integrations.

## Should the hosted dashboard be built now?

Not first. The public configs, docs, and self-hosted workflow should stabilize before a hosted dashboard becomes the main effort.

## Are the files in `old/` canonical?

No. They are archive material and preliminary drafts.

## Which document should guide technical direction?

Start with [../OPENCLAW_FINAL_SPEC.md](../OPENCLAW_FINAL_SPEC.md), then use [roadmap.md](roadmap.md) and [open-core-model.md](open-core-model.md) for public packaging decisions.
