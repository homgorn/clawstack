WordPress path

Static first, WordPress later if you need a CMS or marketing workflow.

Why it ports cleanly

The landing is built as flat semantic sections, so it can be moved into a page template, block theme layout, or generated page body through WP-CLI or the REST API.

Recommended sequence

Launch the static version first. Move to WordPress only when editor workflows, SEO plugin operations, or content publishing needs justify the extra layer.

FAQ

Answer the high-intent questions on the page itself.

Is this a free product or a paid product?

The repo is the free product core. Paid offers are setup and managed support layers on top.

Do I need a dashboard immediately?

No. The first frontend should sell the offer clearly. Dashboard work should come later and only if demand proves it.

Can this be moved into WordPress later?

Yes. The content blocks are designed to move cleanly into WordPress via a template, WP-CLI, or the REST API.
